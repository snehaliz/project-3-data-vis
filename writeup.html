Project Write-Up

Rationale for Design Decisions

Our group sought to display the growth of Costco across the United States, over time. There are two main components to display this data-- geographic location and time data. We have an interactive bar where a user can scrub through dates from 1978 to 2020 that will display all the Costcos opened on or before this date. If the user does not interact with the time bar, it will automatically proceed in a chronological order. Our encodings are as follows:

  • Mark: Circle
  • Color: Red
  • X-axis: Longitude (Q-interval)
  • Y-axis: Latitude (Q-interval)
  • + Mark: Text
  • X: Month, Year (Ordinal)
  • + Mark: US map
  • Color: Grey
  • X: Longitude (Q-interval)
  • Y: Latitude (Q-interval)

While just printing out the coordinates of each Costco presents a map that looks similar to the map of the US, we decided that overlaying this data on the map of the US is much more granular, explicit, and easier to interpret where certain major cities lie. However, the state lines are only included for the 48 continental states while Hawaii and Alaska Costcos are offset to the bottom left of the map, which is common in many maps. We also included a description at the top to be exceptionally clear. If we had more time for development, we would attempt to implement a map that also includes Alaska and Hawaii. Our original data included international Costcos, but we decided to focus on the US since it was the highest abundance of Costcos ranging over the longest period of time. A map of the entire world would be difficult to inspect on only one web page.

We decided to map the circles for each Costco red as that corresponds to the logo of Costco. The color red is striking against the grey map.

Lastly, as our main visualization is based on the chronological spread of Costco, we thought it was most fitting for the user to be able to interact with the time variable. This allows the user some insights into how the company spread both geographically and chronologically.

Overview of Development Process

Overall, our group was able to collaborate effectively and efficiently!

We came up with an idea for our visualization quickly, especially due to the help of the examples on the D3 website. Our two main challenges while working on this project were (a) properly loading in and accessing our data from a CSV file and (b) making sure the circles on the map properly appeared in correspondence with the date bar. As for challenge (a), D3 provided quite good examples, but had data in a different format, so we needed to figure out how to change the data loading and properly access our CSV file. We used a lot of console.log() statements to help with debugging and make sure we were properly accessing each variable. For challenge (b), we realized we had to filter our data points each time we updated the date for them to properly match.
